## Data Pipeline Fix (FanGraphs blocked)

### Problem
FanGraphs returns HTTP 403 on the legacy API (`leaders-legacy.aspx`). pybaseball v2.2.7 is broken for FG data. This killed access to FIP, xFIP, SIERA, SwStr%, F-Strike%, CSW%.

### Fix
- **cache.py**: Added `_fetch_savant_custom()` and `_fetch_savant_custom_with_fallback()` ‚Äî fetches k_percent, bb_percent, whiff_percent (SwStr%), f_strike_percent, csw_rate from `baseballsavant.mlb.com/leaderboard/custom` CSV export (not behind Cloudflare)
- **cache.py**: Added `_pitcher_savant_pitch_cache` global and `get_pitcher_savant_pitch()` lookup function
- **cache.py**: Updated `refresh_daily_caches()` to fetch the new Savant pitch-level cache
- **cache.py**: Fixed `_blend_savant_rows()` ‚Äî when PA data is missing (Savant custom leaderboard has no PA column), defaults to 50/50 current/prior year blend instead of returning current-year-only
- **cache.py**: Added `whiff_percent`, `f_strike_percent`, `csw_rate` to blend_cols in `_blend_savant_rows()`
- **cache.py**: Both blend functions now treat NaN as missing (`pd.isna()` check) instead of treating it as a valid value
- **pitcher_stats.py**: Added `get_pitcher_savant_pitch` import and fallback block ‚Äî fills SwStr%, F-Strike%, CSW%, K%, BB% from Savant when FanGraphs is unavailable
- **pitcher_stats.py**: Added `_calc_fip()` function ‚Äî calculates FIP from MLB API counting stats (13*HR + 3*(BB+HBP) - 2*K) / IP + constant. Only runs when IP >= 10 (below that, FIP is noise)
- **pitcher_stats.py**: Added xERA as xFIP proxy when FanGraphs xFIP unavailable
- **pitcher_stats.py**: Stores `_hr`, `_bb`, `_hbp`, `_so` counting stats from MLB API for FIP calculation
- **Installed**: `curl-cffi` pip package (attempted for FG bypass, ended up not needed but still installed)

## Model Logic Changes

### confidence.py
- Added `early_season` penalty (-10 points) to both `calc_k_confidence` and `calc_nrfi_confidence` ‚Äî fires when pitcher has < 20 IP in current season

### strikeout_predictor.py
- **Walk rate bonus**: Added upper cap `min(0.5, ...)` ‚Äî was `max(-0.5, ...)` with no ceiling
- **Pitch count factor (W_PITCH_COUNT=0.10)**: Changed from `expected_batters * (k_rate / 100)` (duplicate of season K-rate factor) to `expected_batters * (LEAGUE_AVG_K_RATE / 100)` ‚Äî now uses league-average K rate as a neutral workload baseline
- **Opposing K-rate factor (W_OPP_K_RATE=0.15)**: Changed base from `(season_ks or csw_ks or 6)` to `expected_batters * (LEAGUE_AVG_K_RATE / 100)` ‚Äî eliminates double-counting of season K rate
- **TTOP adjustment**: Moved from after market blending to before it ‚Äî model's TTOP skepticism is now baked into the pure projection before 60/40 blend with market line
- Added `early_season` data flag (IP < 20)

### nrfi_predictor.py
- **Variable rename**: `leadoff_obp` ‚Üí `leadoff_ops` (was holding OPS data, named as OBP)
- **Team offense signal**: Changed from `away_batting_top[0].get("ops")` (leadoff only) to average of top 3 batters
- **Leadoff K-rate flag**: Changed from OR to AND ‚Äî requires both sides to have K-rate data for +7 confidence
- **NRFI rate**: Now adjusts probability (up to +4% boost for rates > 70%, -3% penalty for rates < 40%), not just confidence scoring
- Added `early_season` data flag

### game_predictor.py
- **Barrel rate**: Now contributes to run differential at 0.15 weight ‚Äî was computed but only used as a risk flag
- **Park weight**: 0.10 ‚Üí 0.08 (partially baked into pitcher ERA already)
- **Weather weight**: 0.10 ‚Üí 0.05 (noisier signal)
